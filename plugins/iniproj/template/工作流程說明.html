Step by step

開新案 · 產出規格初稿

照順序做就好。左邊是「你做什麼／你說什麼」，Claude 會接著做事——你只負責決策，不負責動工。

1

安裝 plugin 每人一次

你做在 Claude Code 輸入 /plugin marketplace add shellsirius/iniProj-plugin → /plugin install iniproj → /exit 重開生效。
為什麼裝好後，「開新專案」這句話在任何資料夾都能用——這是整套流程的入口。只需做這一次。
2

說「開新專案」，長出一整包 開新專案 skill

你說「開新專案」。
Claude問你目標路徑與專案名（例 proj-某某案）→ 把整包範本複製過去。骨架包裡已備好所有規則與工具。
🪄 你不用自己複製資料夾了。一句話，Claude 幫你在指定位置長一包乾淨的新案。
3

站進新案資料夾，再啟動 Claude 終端機

你做終端機 cd 進剛生出的新案資料夾 → 輸入 claude 啟動（或 VS Code 開該資料夾）。
🚪 先走進房間，再請人進來。分析 RFP、產 PRD／SA 的 skill 綁在新案資料夾裡，Claude 得「站進去」才讀得到。
4

放入售前文件 PM/input/

你做把 RFP／服務建議書／合約丟進 PM/input/。
格式.docx .pdf .txt .md 都行，可一次放多份。
5

跟它說「importRFP」 importRFP skill

你說「importRFP」（或「分析這份 RFP」）。
Claude讀文件 → 產 gap-analysis.md（哪些合約已綁定、哪些待你決策，附出處）→ 產 prefill.js → 自動開啟新手包問答網頁。
6

補完問卷，收回產出 PM/output/iniRFP.md

你做網頁裡「合約已綁定」的題目已自動帶出並標 📎；補完「待決策」題目 → 按下載 .md。
你說「填好了」。
Claude把下載的 .md 搬成 PM/output/iniRFP.md——這份是 RFP 規格初稿，可能還不完整，後面的文件都以它為依據。
Generate

產 PRD · 生成需求文件

規格初稿到位後，一句話讓 Claude 把它變成結構化的需求檔。

7

跟它說「產PRD」 iniRFPtoPRD skill

你說「產PRD」（或 iniRFPtoPRD）。
Claude讀 iniRFP.md → 依規範產出 PRD R檔 → 更新階段索引 PRD.md → 生成追溯總表 traceability.md。動工前它會先列「要產哪幾筆需求」給你確認。
📝 PRD 也能從會議記錄來——見下一站「會議記錄 → PRD」。
Another source

會議記錄 → PRD

PRD 不一定要從問卷來——需求訪談、客戶會議的記錄，也能直接變成需求檔。

📝

跟它說「會議產PRD」 MeetingtoPRD skill

你做把會議／訪談逐字稿放進 PM/Meeting/。
你說「會議產PRD」，或指定檔案 MeetingtoPRD PM/Meeting/某次會議.md。
Claude萃取需求（附逐字稿佐證讓你核對）→ 判定是新增還是修訂既有需求 → 先列清單給你確認 → 寫入 PRD。
💡 一次處理一份會議。來源可能是需求訪談紀錄（多半新增需求），也可能是後續會議（修訂既有需求）——Claude 會逐項判定、先列清單給你確認。
Next layer

產 SA · 把需求拆成功能

需求定了，下一層是「怎麼拆」——SA 把每條需求拆成功能與模組，之後 SRS 再逐功能寫頁面規格。

🧭

跟它說「產SA」 PRDtoSA skill

你說「產SA」（或 PRDtoSA）。
Claude讀 PRD R檔 → 把需求拆成功能、配編號（如 BO-1-2）、定模組 → 先列功能／模組規劃給你確認 → 寫模組 SA、登錄 SA.md、回填追溯表。
🧭 SA 只談「怎麼拆、誰依賴誰」，不碰頁面長相——那是之後的 SRS。
Where things land

東西都跑去哪了

跑完一輪，新案資料夾長這樣。你丟進去的、Claude 產出來的，一眼對照。

proj-某某案/
├─ PM/
│   ├─ input/          # ← 你丟售前文件（RFP／合約）
│   ├─ Meeting/        # ← 會議／訪談記錄（PRD 的另一種來源）
│   ├─ output/iniRFP.md  # ← 問答產出的規格初稿
│   └─ index.html      # 新手包問答精靈
├─ PRD/
│   ├─ R0001.md …     # 一筆需求一個檔
│   ├─ PRD.md          # 階段索引
│   ├─ traceability.md # 追溯總表（自動生成）
│   └─ rules/          # PRD 撰寫規範
└─ SA/
    ├─ SA.md           # 功能架構索引（需求拆成功能）
    ├─ {系統}/…        # 模組層功能分析
    └─ rules/          # SA 撰寫規範
Keep in mind

三個小提醒

新手最常踩的三件事，先記著就不會卡。

🚪

站對資料夾

開新專案 在哪都能用；但 importRFP 等要先 cd 進新案、最後才 claude。沒觸發多半是站錯地方——/exit 重來即可。

🤔

不確定就不硬填

合約沒寫死的，標「待確認」；Claude 也不會腦補，模糊的它會問你，而不是自己編。

🔒

版控要你開口

git 的 commit／push 只有你明說才做。平常 Claude 只改檔＋給你看，不會自己上傳。
