# SRS 元件庫規則

欄位規格表的「元件類型」「資料格式（驗證）」「資料來源」一律依本檔，**不自由發揮**，確保跨頁一致。

---

## 一、元件詞庫（「元件類型」欄只能用這些詞）

| 元件 | 用途 |
|---|---|
| Text Field | 單行文字輸入 |
| Password Field | 密碼輸入 |
| OTP Field | 驗證碼（多格、每格單碼） |
| Textarea | 多行文字 |
| Rich Text Editor | 富文本編輯 |
| Select | 固定選項單選下拉 |
| Combobox | 可搜尋下拉 |
| Date Picker | 日期（格式 YYYY/MM/DD；可「（區間）」或「（單選）」） |
| Number Field | 數字輸入 |
| Checkbox | 多選框 |
| Radio Group | 單選按鈕群 |
| Switch | 二態開關 |
| File Upload | 檔案上傳 |
| Lookup Button | 開 Dialog 選取（於「資料格式」描述按鈕行為） |
| Tag | 標籤 |
| Badge | 狀態徽章（列表狀態值用 `{ 選項A / 選項B }` 列舉） |
| Image | 圖片顯示 |
| Button | 按鈕、操作圖示 |
| Alert / Alert Dialog / Toast / Tooltip | 訊息類 |
| 純文字（唯讀） | 唯讀欄位 |

> 列表的「資料格式」型別用：Text、Number、Date、Tag、Badge、Image、Button、Icon。

---

## 二、驗證規則固定措辭（寫在「資料格式」欄，多條用 `a. b. c.` ＋ `<br/>`）

- **必填**：離開元件時，若為空值則即時顯示錯誤提示：此欄位為必填。
- **字元數上限**（Text Field）：若輸入超過最大字元數，需即時顯示錯誤提示：長度最長 N 字。
- **限制輸入**：限制輸入：僅能輸入半形數字／英數字／…（依需要）。
- **唯一性檢核**：點擊儲存時，若與既有 XX 重複，顯示錯誤提示：XX 已存在。
- **Email 格式**：採固定規則組（見 §2.1）。

> 驗證規則**不另開欄**，一律寫在「資料格式」欄。

### 2.1 Email 格式

Email 欄位採下列固定驗證規則組，各 SRS 於該欄之「資料格式」逐條列出：

```
a. 須符合 Email 格式「帳號@網域」
b. 系統自動去除前後空白
c. 不允許空白、全形字元及不合法特殊符號（詳本節「輸入限制檢核項目表」）
d. 若有填寫內容且離開元件時，若格式錯誤則即時顯示錯誤提示：Email 格式不正確
e. 如需檢查 Email 是否重複，Email 比對將不區分大小寫
```

**輸入限制檢核項目表**

| 檢核項目 | 規則 |
|---|---|
| 帳號段（@ 前方） | 僅允許 A-Z、a-z、0-9、.、_、-、+ |
| 網域段（@ 後方） | 僅允許 A-Z、a-z、0-9、.、- |
| 點號限制 | 不可連續出現 .. |
| 點號位置 | 帳號段與網域段不可由 . 開頭或結尾 |
| 減號位置 | 網域段每一層不可由 - 開頭或結尾 |
| @ 符號 | 僅可出現一次 |
| 全形符號 | 不允許，例如 ＠、．、＿ |
| 中文字元 | 不允許 |
| 空白字元 | 不允許 |

> Email 欄位最大字元數預設 **50**；各 SRS 於該欄之「最大字元數」直接填入實際值。

## 三、資料來源四分法（「資料來源」欄只填這四類）

| 類別 | 寫法 |
|---|---|
| 相依模組／功能 | `來源 XX-X-X`（如 `來源 FB-1-3`） |
| 系統固定選項 | `系統固定選項` |
| 使用者輸入 | `使用者輸入（XX-X-X-M##）`，註明首次鍵入頁 |
| 查詢條件暫態 | `使用者輸入`（暫態篩選，不落資料） |

---

## 四、個資遮罩（選配）

- 含個資之頁面，於欄位表「備註」欄**前**插入「遮罩」欄，需遮罩者標 `＊`。
- 配套「取消遮罩」Dialog 揭露明文（行為於該頁 UI flow 描述）。
