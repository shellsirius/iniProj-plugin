# SRS 檔案拆分與分層規則

> 規範 SRS 資料夾分層、單一功能頁過大時的**子頁**拆分與命名。

---

## 一、分層

```
SRS/{系統}/{模組}/{功能}.md      例 SRS/FB/FB-1/FB-1-2.md
```

- 模組資料夾純編號（`FB-1/`），對應 SA 模組；不帶模組中文名。
- 一個 SA 功能 ＝ 一個主檔；其子頁放同一模組資料夾下，**不再加資料夾層**。

---

## 二、何時拆子頁

一功能＝一頁檔；該頁過大時拆**子頁**。下列任一條件成立即可拆：

- 頁面區塊 ≥ 5（查詢／列表／多個表單 Dialog…）。
- 整檔 ≥ 200 行。
- 明細分頁（tab）> 2。
- 單一區塊規格 ≥ 80 行。

不符則維持單檔。

---

## 三、主檔／子頁分工

- **主檔** `{功能編號}.md`：頁面區塊索引（連回各子頁）、**完整頁內 UI Flow**、跨子頁備註；無歸屬的小區塊可直接留主檔。
- **子頁** `{功能編號}-{分組名}.md`：該分組的區塊／欄位規格；內部切換補述，**不重畫主流程圖**。
- 子頁頂部固定引述：

  ```markdown
  > 本檔為 {功能編號} {功能名}「{分組名}」之頁面規格；完整區塊與 UI Flow 詳 [{功能編號}.md]({功能編號}.md)。
  > 含區塊：{區塊1}、{區塊2}…
  ```

---

## 四、命名

- **主檔**：`{功能編號}.md`（如 `FB-1-2.md`）。
- **子頁**：`{功能編號}-{分組名}.md`（如 `FB-1-2-列表與新增.md`、`FB-3-1-基本資料.md`）。
  - 分組名用**業務語彙**繁中（反映畫面範圍），與功能地圖／SA 語彙一致；**不用** M／區塊範圍命名。
  - 子頁**沿用同一功能編號**，不另開編號；SA 可回註拆分。
