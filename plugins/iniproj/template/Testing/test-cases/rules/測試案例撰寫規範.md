# 測試案例撰寫規範

規範 iniProj 骨架包中 `Testing/test-cases/` 的測試案例文件。測試案例是 SRS／PRD 下游的**驗證層**，回答「**怎麼測、預期什麼**」。

> **流向**：PRD（要什麼）→ SA（怎麼拆）→ SRS（畫面長怎樣）→ **測試案例（怎麼驗證）**。
> 一筆測試案例（TC）驗證某 SRS 功能是否符合 PRD 需求與驗收條件。

---

## 一、定位

| 層 | 回答 | 來源 |
|---|---|---|
| **測試案例** | **怎麼測、預期結果是什麼** | PRD R檔（§三 情境／§四 業務規則／§六 驗收條件）＋ 追溯到的 SRS 頁面（步驟／欄位／驗證） |

- TC「**要驗什麼**」從 **PRD R檔**取：§三 使用情境、**§四 業務規則與限制（反向／邊界／卡控案例）**、§六 驗收條件。
- TC「**怎麼操作**」從**追溯到的 SRS 頁面**取：操作步驟、欄位、驗證、訊息細節。
- 案例需涵蓋**正向＋反向／邊界**；不重寫 PRD 業務動機、不重寫 SRS 完整 spec；聚焦**可執行、可判定通過與否**的場景。

---

## 二、資料夾與檔名

```
Testing/
├── test-cases/
│   ├── {階段}/                          # 依 PRD 階段分資料夾，如 1-A、1-B
│   │   └── testing-cases-{階段碼}-{R編號}.md   # 階段碼去連字號，如 1-A→1A
│   └── rules/測試案例撰寫規範.md         # 本檔
└── 測試案例-範本.md
```

- **一個 R檔一個測試案例檔**；同一 R 跨多階段 → 各階段資料夾各自成檔。
- 檔名例：`test-cases/1-A/testing-cases-1A-R0001.md`。

---

## 三、案例編號（TC）

- 格式 `TC{三位以上連續數字}`（如 `TC001`、`TC042`）。
- **全域唯一、不回收**：新增時取既有最大號 +1，跨檔不重複。

---

## 四、測試案例表（15 欄，固定順序）

| 欄位 | 必填 | 說明 |
|---|---|---|
| 需求編號 | ✓ | 對應 PRD 的 R 編號；多筆以逗號分隔 |
| 需求說明 | ✓ | 該需求核心目的，簡述（可含【系統別】前綴） |
| 測試功能項目 | ✓ | SRS 代碼與功能名，全形斜線分隔（如 `FB-1-1 ／ 登入`） |
| 測試屬性 | ✓ | `原本需求` 或 `SRS擴充` 二選一 |
| 案例編號 | ✓ | TC 全域唯一編號 |
| 測試情境／案例名稱 | ✓ | 一句話描述此案例驗證的場景 |
| 測試頁面 | ✓ | `【{系統別}】{頁面路徑}` |
| 前置條件 | 條件 | 執行前需成立的狀態（角色／資料／權限）；可串接其他 TC（如「已操作【TC001】成功」） |
| 其他參考條件 | 條件 | 元件預設值、規則限制等補充；無則填「無」 |
| 測試步驟 | ✓ | 條列操作，編號一致（`1、2、3、`），多行用 `<br>` |
| 測試資料 | 條件 | 步驟用到的輸入／帳號／檔案；無則填「無」 |
| 預期結果 | ✓ | 對應步驟的系統行為與畫面回應（成功訊息／錯誤提示／資料變化），可用 📍 標重點 |
| PC | ✓ | 桌機適用性：`適用`／`不適用` |
| Pad | ✓ | 平板適用性：`適用`／`不適用` |
| Phone | ✓ | 手機適用性：`適用`／`不適用` |

> 平台適用性（PC／Pad／Phone）依該系統的目標載具決定；後台多半僅 PC 適用。

---

## 五、撰寫慣例

- **語系**：繁體中文。
- **顆粒度**：一筆案例聚焦單一場景（新增／編輯／刪除／查詢／放棄／驗證失敗…）；複雜流程拆多筆，以前置條件串接。
- **測試步驟**：純操作動作；**預期結果**帶畫面／訊息／資料變化細節，可被判定通過與否。
- **測試資料**：具體（如「輸入長度超過 10 字」），不泛稱「錯誤資料」。
- **不腦補**：SRS／PRD 未明處（欄位、訊息文案、驗證）標「待釐清」，不自行編造預期結果。
- **追溯**：每筆 TC 必填「需求編號」與「測試功能項目」（SRS 代碼），形成 R ↔ SRS ↔ TC 鏈。

---

## 六、CRUD 標準情境（母版，依頁面取用）

新增／編輯／刪除／查詢類頁面，至少涵蓋下列場景，將 `xx` 換成實體名稱：

- **新增**：必填皆填→成功；必填缺漏→錯誤提示；唯一性重複→錯誤提示；放棄新增→不建立資料。
- **編輯**：載入既有值；修改存檔→成功；放棄編輯→不變更。
- **刪除**：刪除確認→成功移除；有卡控條件→阻擋並提示。
- **查詢**：條件命中→列表顯示；查無→空資料提示；排序／分頁行為。

> 母版為起點，實際以該頁 SRS 與 PRD 驗收條件補齊專屬場景。
